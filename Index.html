A 3-level self-evaluation designed for beginners with no technical background. Take each level honestly — the goal is self-discovery, not a score.

Honour system: Do not Google answers. Do not ask anyone. If you are stuck, sit with the problem — the struggle itself is part of the test. Time taken and approach matter more than being right.

Level 1 — Beginner: Setup, Observation & Basic Thinking

You will install software, perform basic computer tasks, answer scenario questions, and solve simple math. No prior knowledge required — but you will need patience and the ability to follow instructions carefully.

1–2 days Pass: 70% MCQ + Tasks
A
Software Installation & Setup
Tasks — Do & Report
A1. Install PostgreSQL on your computer
Hands-on Task
Go to postgresql.org → download the installer for your operating system (Windows/Mac/Linux). Install it with default settings. During setup, you will be asked to set a password for the default user (postgres). Write that password somewhere safe.

After installation, open pgAdmin (it installs along with PostgreSQL). If you can see the pgAdmin dashboard, you have succeeded.
A2. Install Power BI Desktop
Hands-on Task
Go to microsoft.com/en-us/power-platform/products/power-bi/downloads and download Power BI Desktop (it is free). Install it. Open the application and note what you see on the first screen.

If you use a Mac: Power BI Desktop is only available on Windows. Install it on a Windows machine or use a free virtual machine (search: "how to run Power BI on Mac using VirtualBox" — read 3 articles and summarize the options below).
A3. Basic OS tasks
Hands-on Task
Complete all of the following on your computer:

(a) Create a folder called DataCareer_Evaluation on your Desktop
(b) Inside it, create 3 sub-folders: Level1, Level2, Level3
(c) Open Notepad (Windows) or TextEdit (Mac), type your name and today's date, and save it as my_notes.txt inside the Level1 folder
(d) Take a screenshot of your folder structure and save it in Level1
(e) Find out: How much free storage space does your computer have? Write the answer in my_notes.txt

You will work with data tables, understand SQL logic without writing code yet, frame business problems, and work through an analytical mini-project. This level tests structured thinking.

1–2 days Pass: 70% Scenario-heavy

Guided Technical Task
I1. Create your first database and table
Hands-on Task
Open pgAdmin. Right-click on "Databases" → Create → Database. Name it evaluation_db.

Open the Query Tool (Tools menu → Query Tool). Type and run this exact SQL:

  name VARCHAR(100),
  department VARCHAR(50),
  salary INTEGER
);

INSERT INTO employees VALUES ('Ravi Kumar', 'Sales', 35000);
INSERT INTO employees VALUES ('Priya Nair', 'HR', 28000);
INSERT INTO employees VALUES ('Arjun Shah', 'IT', 52000);

SELECT * FROM employees;
If you can see the 3 rows appear at the bottom, you have run your first SQL query. Take a screenshot.

Level 3 — Advanced: End-to-End Mini Project, Analytical Depth & Career Clarity

You will complete a real mini-project, write a short analytical report, evaluate a dashboard, and honestly assess your own readiness. This level determines career suitability with clear guidance.

1 full day Pass: 65% Project-based
J
Mini Analytical Project — End to End
2–4 hour project
Project Brief

A small restaurant chain with 3 branches (Hyderabad, Bangalore, Chennai) wants to understand their performance. You are given the following monthly data for the last 6 months. Your job is to analyze it and provide 3 clear recommendations.

J1. Build the dataset in Excel
Hands-on Task
Create a new Excel file called Restaurant_Analysis.xlsx. Enter this data:

Month | Branch | Total Orders | Revenue (₹) | Customer Complaints | Staff Count
Jan | Hyderabad | 1200 | 4,80,000 | 18 | 12
Jan | Bangalore | 950 | 3,80,000 | 7 | 10
Jan | Chennai | 800 | 3,20,000 | 5 | 8
Feb | Hyderabad | 1350 | 5,40,000 | 22 | 12
Feb | Bangalore | 1100 | 4,40,000 | 9 | 10
Feb | Chennai | 720 | 2,88,000 | 3 | 8
Mar | Hyderabad | 1100 | 4,40,000 | 30 | 11
Mar | Bangalore | 1250 | 5,00,000 | 6 | 11
Mar | Chennai | 900 | 3,60,000 | 4 | 9
Apr | Hyderabad | 980 | 3,92,000 | 35 | 10
Apr | Bangalore | 1300 | 5,20,000 | 5 | 11
Apr | Chennai | 950 | 3,80,000 | 6 | 9
May | Hyderabad | 870 | 3,48,000 | 42 | 9
May | Bangalore | 1400 | 5,60,000 | 4 | 12
May | Chennai | 1000 | 4,00,000 | 5 | 10
Jun | Hyderabad | 760 | 3,04,000 | 51 | 9
Jun | Bangalore | 1500 | 6,00,000 | 3 | 12
Jun | Chennai | 1050 | 4,20,000 | 4 | 10

Now calculate for each branch: Total 6-month Revenue, Average Monthly Orders, Average Complaints. Use Excel formulas.
J2 — Based on the project data
Looking at the Hyderabad branch data from January to June, what is the most concerning trend?
J3 — Analytical reasoning
In June, Hyderabad had 51 complaints from 760 orders. Bangalore had 3 complaints from 1500 orders. What is Hyderabad's complaint rate and what does that tell you?
J4. Create a dashboard in Power BI
Hands-on Task
Load your Restaurant_Analysis.xlsx into Power BI Desktop. Create a report with at least 3 visuals:

(a) A line chart showing Revenue by Month for each Branch (3 lines)
(b) A bar chart showing Total Complaints by Branch
(c) A table or card showing the best-performing branch (by revenue)

Format your report so it looks presentable. Add a title. Save it as Restaurant_Dashboard.pbix.
J5. Write your analytical report
Written Deliverable
Open a Word document or Notepad. Write a 1-page report (200–300 words) addressing:

(a) Which branch is performing the best and why
(b) Which branch needs immediate attention and what specific problem do you see in the data
(c) Give 3 specific, data-backed recommendations to the restaurant owner

Save it as My_Analysis_Report.txt in your Level3 folder.

Note: Your recommendations must reference actual numbers from the data — not vague suggestions like "improve customer service."
